# spark-self-heal

Self-healing data pipeline on AWS (Spark + Glue + Athena + Airflow) with an
AI agent that diagnoses failures and proposes patches via pull request.

## Status

Work in progress — Phase 2 (PySpark "victim" pipeline) complete; Phase 3
(Airflow orchestration + structured logs) is next.

## Stack

- **AWS**: S3, Glue (Catalog + Jobs), Athena, CloudWatch Logs
- **Orchestration**: Apache Airflow (self-hosted via docker-compose)
- **Infrastructure as Code**: Terraform
- **Agent**: Python + Anthropic Claude API + tool-use loop
- **CI/CD**: GitHub Actions

## Domain

Fintech payments. The pipeline ingests JSON transaction events, validates
them, and writes partitioned Parquet to a silver zone queryable via
Athena. See `docs/failure-modes.md` for the failure taxonomy the agent
learns to diagnose.

## Quick start (local)

```bash
# 1. Python venv + dataset generator
python3 -m venv .venv && source .venv/bin/activate
pip install faker
python scripts/generate_dataset.py

# 2. Provision AWS resources (requires aws CLI profile + ~/.aws/credentials)
cd terraform
terraform init
terraform apply
```

See `docs/adr/003-cost-guardrails.md` before running `terraform apply`.

## Architecture decisions

See `docs/adr/` for the full ADR log. Highlights:

- **ADR-001**: Real AWS account instead of LocalStack (Glue/Athena coverage)
- **ADR-002**: Three agent autonomy modes (`observe` / `propose` / `apply`)
- **ADR-003**: Cost guardrails — $10/month budget, tagging, single region
